# Points Bot 🏆

بوت تليجرام لنظام نقاط ومستويات: أي حد يرد على حد تاني برد بيمثل "مساعدة حقيقية"
(البوت بيتأكد من ده عن طريق Gemini)، ياخد نقاط، ولما نقاطه توصل لحد معين ياخد
تاج مستوى جديد من **L1** لغايه **L10**.

## ⚠️ مهم جداً قبل ما تشغّله

البوت ده شغال بطريقة **Cron** (يتشغل، ياخد الرسايل الجديدة، يقفل) — مش
Long Polling مستمر. عشان كده **لازم تتأكد إن مفيش نسخة تانية من البوت شغالة
بنفس التوكن في نفس الوقت** (مثلاً نسخة قديمة بتعمل `bot.launch()` في ريبو أو
سيرفر تاني)، وإلا هيرجع يحصل 409 Conflict تاني. لو عندك الـ workflow القديم
بتاع Ai_point1_bot شغال، وقفه أو امسحه الأول.

### التاج جنب الاسم — من غير ترقية العضو خالص

تليجرام ضاف method جديد اسمه `setChatMemberTag`: بيدي لقب (Tag) لعضو
عادي في الجروب **من غير ما يترفع أدمن خالص**. العضو بيفضل عضو عادي 100%،
اللي بيتغير بس هو إن اسمه يظهر جنبه لقب (زي L1، L2... إلخ).

الشرط الوحيد: **البوت نفسه** (مش العضو) لازم يبقى أدمن في الجروب وعنده
صلاحية واحدة بس اسمها **"Edit Member Tags"** (`can_manage_tags`). من
غيرها البوت مش هيقدر يحط أي تاج.

خطوات التفعيل:
1. ضيف البوت أدمن في الجروب.
2. من صلاحيات الأدمن بتاعته، فعّل **"Edit Member Tags"** بس (تقدر تسيب
   باقي الصلاحيات كلها مطفية — حذف رسائل، حظر، تعديل جروب... البوت
   مش محتاجهم).
3. خلاص، أي عضو ياخد تاج جديد هيظهر جنب اسمه في قايمة الأعضاء من غير
   ما يترفع أدمن ولا ياخد أي صلاحية.

### ⚠️ ملاحظة عن الإيموجي المخصص (Custom Emoji)

حسب توثيق تليجرام الرسمي، إرسال Custom Emoji عن طريق بوت شغال **بس** لو:
- صاحب البوت (اللي عامل التوكن من BotFather) عنده **Telegram Premium**، أو
- البوت مشتري يوزرنيم إضافي من Fragment (حل مكلف وغير عملي هنا).

لو صاحب حساب البوت مش عنده Premium، الإيموجي المخصص هيتحول تلقائي
للإيموجي البديل (🎉 و🏆) اللي حاطينهم كـ fallback في الكود.

## أوامر الأونر

الأوامر دي متاحة بس للأونر (ID: `8355232956`). أي حد تاني يستخدمها بيرجع
له رسالة "الأمر ده للأونر بس".

كل الأوامر (ما عدا `/leaderboard` و`/ownerhelp`) لازم تتبعت **كرد** على
رسالة الشخص المطلوب:

| الأمر | الوظيفة |
|---|---|
| `/addpoints [عدد]` | إضافة نقاط (افتراضي 1 لو مكتبتش رقم) |
| `/removepoints [عدد]` | خصم نقاط (متنزلش تحت الصفر) |
| `/setpoints [عدد]` | تثبيت الرصيد على رقم معين بالظبط |
| `/resetuser` | تصفير رصيد الشخص بالكامل (مش بيشيل التاج تلقائي) |
| `/setlevel [1-10]` | تثبيت مستوى معين مباشرة (بيحسب النقاط المطلوبة تلقائي) |
| `/removetag` | شيل التاج من الشخص (بيفضل عضو عادي زي ما هو) |
| `/userstats` | عرض نقاط/مستوى/عدد مساعدات الشخص (لو من غير رد، بيوريك بياناتك انت) |
| `/leaderboard` | أعلى 10 في الجروب (من غير رد) |
| `/ownerhelp` | يوريك الأوامر دي كلها |

## طريقة كشف "الشكر"

لو حد رد على حد بكلمة شكر بسيطة (زي "شكراً"، "تسلم"، "يعطيك العافية"...
في جملة قصيرة)، البوت بيتعرف عليها بـ Regex بسيط من غير ما يستهلك طلب
Gemini، وبيدي النقطة **لصاحب الرسالة الأصلية** (اللي اتشكر) مش للشخص
اللي بيشكر. لو الرسالة طويلة وفيها شرح فعلي مع الشكر، بتروح للتقييم
العادي بتاع Gemini عشان تتقيم كإجابة.

## الإعداد

1. اعمل ريبو جديد على GitHub وارفع الملفات دي فيه.
2. من **Settings → Secrets and variables → Actions** ضيف:
   - `BOT_TOKEN`: توكن البوت من BotFather
   - `GEMINI_API_KEY`: مفتاح Gemini API
3. تأكد إن الـ workflow عنده صلاحية الكتابة:
   Settings → Actions → General → Workflow permissions → اختار
   **Read and write permissions**.
4. البوت هيتشغل تلقائي كل 5 دقايق (`.github/workflows/bot.yml`)، أو تقدر
   تشغّله يدوي من تاب **Actions → Points Bot Runner → Run workflow**.

## طريقة عمل التقييم

- البوت بيشتغل بس لما رسالة تكون **رد** على رسالة تانية.
- بيبعت الرسالتين لـ Gemini، وهو بيحدد:
  - هل الرسالة الأصلية أصلاً سؤال/مشكلة محتاجة مساعدة؟
  - هل الرد فعلاً بيساعد؟ وبيدي جودة من 1 لـ 3 (وده اللي بيحدد عدد النقاط).
- لو مفيش مساعدة حقيقية (زي "تمام" أو رموز تعبيرية) البوت مبيديش نقاط.

## الملفات

- `index.js` — السكريبت الأساسي اللي بيتشغل كل مرة (getUpdates → معالجة → حفظ).
- `telegram.js` — كل دوال الاتصال بـ Telegram Bot API (إرسال رسائل، تاج الأعضاء...).
- `points.js` — منطق إضافة النقاط + رسالة التهنئة + التاج عند صعود مستوى.
- `thanks.js` — كشف رسائل الشكر البسيطة (Regex) بدل استهلاك Gemini.
- `commands.js` — أوامر الأونر (إضافة/خصم نقاط، تثبيت مستوى، ليدربورد...).
- `gemini.js` — الاتصال بـ Gemini API لتقييم هل الرد إجابة/شرح حقيقي.
- `levels.js` — جدول المستويات وحساب النقاط.
- `db.js` — قراءة/كتابة `data.json` (نقاط المستخدمين) و `offset.json` (آخر رسالة اتعالجت).
- `.github/workflows/bot.yml` — الـ workflow اللي بيشغل السكريبت ويعمل commit للبيانات.
